Roles & Permissions – Concept Overview

1. Core Idea

The system separates roles and permissions.

* Roles describe what a user is
* Permissions define what a user can do

A user is not limited by a single role.
A user can have multiple roles and additional permissions.

⸻

2. Roles

Roles are simple labels used to organize users.

Examples:

* SuperAdmin
* LeagueOwner
* LeagueAdmin
* Player

Roles help structure the system, but they do not strictly control actions on their own.

⸻

3. Permissions

Permissions define specific actions.

Examples:

* Manage a league
* Create tournaments
* Invite players
* Approve join requests
* Update match scores
* Edit own profile

Permissions are the true source of control in the system.

⸻

4. Relationship Between Roles and Permissions

* Roles group common permissions together
* Permissions can also be assigned directly to a user
* A user can gain abilities without changing their role

This allows flexibility without breaking structure.

⸻

5. Flexible User Behavior

A single user can:

* Be an admin and a player at the same time
* Have special permissions without changing roles
* Participate in the league while also managing it

The system does not restrict users to one function.

⸻

6. Scoring Concept

Scoring is not a role.

* It is a permission
* Any user can be allowed to score
* This avoids creating unnecessary roles

⸻

7. League Participation

Users can:

* Join a league by invitation
* Request to join and be approved

Once inside a league, their roles and permissions define what they can do.

⸻

8. Control and Responsibility

* Higher-level users can grant permissions to others
* Permissions can be adjusted without changing roles
* Control is distributed but still structured

⸻

9. Key Principle

The system prioritizes flexibility with control:

* Roles provide structure
* Permissions provide precision

This makes the system scalable, adaptable, and easy to extend.
